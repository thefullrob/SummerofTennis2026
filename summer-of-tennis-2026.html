Ren's plan in one paragraph

Ren (19, ex-Middlebury #3, UTR 8.47) has a strong base and college-match volume to draw from. With limited new-match opportunities this summer, the path to 9.0 runs through three things: weapons sharpening (squeezing more from her best shot), quality hitting against Charlie (he's a 9.67 — that's an above-college-level hitting partner), and match-volume substitution (sets played at home count for tactical sharpness even if they don't move UTR). She'll likely leave for Middlebury in late August, so the plan front-loads her opportunities for hitting with Charlie.

Ren's three focus areas this summer

  1. Hands at the net — soft volleys, drop volleys, angle volleys, swing volleys when the ball sits up. The specific bug to fix: easy volleys missed in the net or wide while moving forward. Most amateur volley misses come from over-swinging while still moving — the fix is "stop, then hit." Block-volley footwork drill is the foundation.
  2. Transition decisions — when to come in — knowing which ball is the approach ball and which one to stay back on. Quick rule of thumb: any ball she takes inside the baseline AND can hit aggressively = approach. Any ball that pulls her wide = stay back. The drill is forcing her to call out loud ("approach!" or "stay!") as she's moving — builds the decision before the execution.
  3. Forehand weapon — maintenance only — her FH is already her best shot. Don't over-drill it. One short FH block (20 min) per session is enough to keep it sharp. The training time freed up goes to net and transition.

UTR math: what 9.67 → 11.0 actually requires

UTR moves on rated singles results, not training. Beating a 10.5 by 6-4 6-2 is worth roughly a 10.8 single-match rating. To pull a 12-match average UTR up by 1.33 points, you typically need 8–10 strong wins over 10.5+ opponents and the avoidance of bad losses (losing to a 9.0 caps your ceiling). Practical implication: at L1 events Charlie's job is to compete — wins move him; losses to higher-ranked players don't hurt him much. At L5/L3 events the job is to not lose to anyone below 9.5. That asymmetry should change his pre-match mindset by event level.

How weights interact with tennis

3×/wk weights on Tue / Thu / Sat only — never on back-to-back days. Muscles need ~48 hours to recover between sessions or you're just digging a fatigue hole. Avoid heavy lower-body the day before competition or before serve-heavy sessions. Suggested split: Tue (full body, moderate), Thu (upper body + rotational core), Sat (lower body + plyo — but skip if it's a tournament Saturday). Sun, Mon, Wed, Fri are never weights days. When a tournament Saturday cancels Sat weights, do NOT shift them to Fri — just drop the session, recovery beats catch-up.

About the projected UTR (Momentum) number

The "projected UTR" on the Today tab is a motivational counter, not a prediction. Real UTR only moves on rated singles match results. Each completed practice + each weights session adds +0.015 projected UTR for each kid — a notional rate that, if they hit the full plan through Aug 16, gets Charlie past 11.0 and Ren past 9.0 with room to spare. The point is to make daily consistency visible: a streak of zeroes shows up immediately; a strong week pushes the bar forward. Use it for momentum, not as a prediction of how Kalamazoo will go.

How to count a session

One practice = a real hitting block (90 min+ counts). Half-effort doesn't count. Weights = the actual lift session, not the warmup. If the kid wants to be honest with the counter, that's the integrity standard.

How to share this with the kids

This page is a single self-contained HTML file. To put it on the web so the kids can click a link, you have three good options ranked by ease:

  1. Netlify Drop (~30 seconds, no account needed): go to app.netlify.com/drop, drag the HTML file onto the page. You get a URL like radiant-otter-abc123.netlify.app instantly. Bookmark it. Re-upload to update.
  2. GitHub Pages (~5 minutes, requires GitHub account, free): create a new repo, upload the HTML as index.html, go to Settings → Pages → set source to main branch root. URL becomes yourusername.github.io/repo-name. To update, push a new file version.
  3. Cloudflare Pages (~10 minutes, requires GitHub): same idea as GitHub Pages but Cloudflare hosts the site. Better speed if the kids check from outside the US. Free.

Important about the check-offs: the gamification state is stored in each browser's local storage. So Charlie checking things off on his phone and Ren checking things off on hers will produce separate counters on each device. The single shared URL is for viewing the plan; the personal progress lives on each kid's own device. That's fine for now — when you want a true shared counter, that needs a backend, which is a bigger project.
